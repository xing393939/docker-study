k8s 四种 service 类型

  1. ClusterIP 是最基础的。
  2. 创建一个 NodePort 类型的 service，Kubernetes 会创建 ClusterIP。
  3. 创建一个 LoadBalance 类型的 service，Kubernetes 会创建 NodePort、ClusterIP。
  4. ExternalName 类型不常用

多集群、多命名空间的管理

  1. kubernetes 中 kubeconfig 的用法
  2. vim ~/.kube/config，修改认证配置
  3. kubectl config get-contexts，获取所有的认证配置
  4. kubectl config use-context kube2，切换到配置名=kube2上
  5. kubectl config rename-context old new，修改配置名
# 设置 KUBECONFIG 的环境变量
export KUBECONFIG=/etc/kubernetes/kubeconfig/kubelet.kubeconfig
# 指定 kubeconfig 文件
kubectl get node --kubeconfig=/etc/kubernetes/kubeconfig/kubelet.kubeconfig
# 使用不同的 context 在多个集群之间切换
kubectl get node --kubeconfig=./kubeconfig --context=cluster1-context

k3s 的一些问题：

  1. 容器的运行环境：docker或者containerd(cri)。k3s默认的容器环境是containerd。
  2. k3s指定容器用docker：
    • curl https://releases.rancher.com/install-docker/19.03.sh | sh
    • curl -sfL http://rancher-mirror.cnrancher.com/k3s/k3s-install.sh | INSTALL_K3S_MIRROR=cn sh -s - --docker
  3. 启动后netstat 看不到80、443端口：nmap -sT -p 80 localhost
  4. 清理没有用到的镜像：k3s crictl rmi --prune
  5. 国内安装脚本：curl -sfL https://docs.rancher.cn/k3s/k3s-install.sh | INSTALL_K3S_MIRROR=cn sh -
  6. 国内修改镜像源：vim /etc/rancher/k3s/registries.yaml 并重启 systemctl restart k3s
    mirrors:
"docker.io":
 endpoint:
   - "https://fogjl973.mirror.aliyuncs.com"
   - "https://registry-1.docker.io"

k3s 集群搭建

# 方法一，安装时指定master
curl -sfL http://rancher-mirror.cnrancher.com/k3s/k3s-install.sh | INSTALL_K3S_EXEC="--node-external-ip=0.0.0.0" sh -s -
curl -sfL http://rancher-mirror.cnrancher.com/k3s/k3s-install.sh | INSTALL_K3S_MIRROR=cn K3S_URL=https://myserver:6443 K3S_TOKEN=${NODE_TOKEN} sh -

# 方法二，先安装，再指定master
curl -sfL http://rancher-mirror.cnrancher.com/k3s/k3s-install.sh | INSTALL_K3S_EXEC="--node-external-ip=0.0.0.0" sh -s -

k3s agent --server https://myserver:6443 --token ${NODE_TOKEN}

如何判断容器环境是docker还是lxc

  1. 如何检测该服务器使用的是Container技术还是VM技术
  2. Golang 如何确定App是否运行在Docker内
  3. docker 如何进入容器：docker exec -it 容器id /bin/bash
  4. kubectl 如何进入容器：kubectl exec -it {pod id} -- /bin/bash
  5. 容器内cat /proc/1/cgroup # 包含docker
  6. 容器内ls /.dockerenv # 如果有就是docker

kubernetes判断cri是docker还是containerd

  1. 通过kubectl describe node命令查看Container Runtime Version项可以得知
  2. 通过kubectl describe pod命令查看Container ID项可以得知
  3. 容器内cat /proc/1/cgroup # 包含docker就是docker，包含kubepods可能是containerd
