docker相关

docker基础

  1. 配置远程2375端口管理，vim /etc/default/docker，加上DOCKER_OPTS="-H tcp://127.0.0.1:2375 -H unix:///var/run/docker.sock"
  2. 更换镜像源，/etc/docker/daemon.json，加上{"registry-mirrors": ["http://hub-mirror.c.163.com"]}
  3. 启动redis：docker run --network subnet01 --name container-redis -p 6379:6379 -d redis --restart=always
  4. 容器开机启动：docker update --restart=always container-redis

docker容器访问宿主机服务（或者其他容器）

  1. docker容器访问宿主机服务
  2. Docker容器间利用bridge网桥实现双向通信
  3. docker四大网络模式
  4. 写host，Dockerfile 添加：ip -4 route list match 0/0 | awk '{print $3 "host.docker.internal"}' >> /etc/hosts
  5. host模式，--network host，使用主机网络作为容器网络
  6. container模式，和一个指定的容器共享IP、端口范围，当然文件系统、进程列表等还是隔离的
  7. bridge模式，在同一网桥的容器处于同一个network：
docker network ls
docker network create --subnet=172.18.0.1/24 subnet01
docker run --name=container-nginx1 --network subnet01 -d image-nginx1
docker run --name=container-nginx2 --network subnet01 -d image-nginx2

docker network create -d bridge my_bridge
docker network connect my_bridge container-db
docker network connect my_bridge container-nginx

volume 和 bind-mount

  1. 跟我一起学Docker——Volume
  2. 数据卷（volume）与捆绑挂载（bind-mount）两个方式都可以与外部共享数据
  3. 数据卷（volume）定义方式：
    1. Dockerfile 中指定 VOLUME /container/dir，其他容器要共享就--volumes-from container-name
    2. docker run -v /host/dir:/container/dir
  4. 捆绑挂载（bind-mount）定义方式：--mount type=bind,source=/host/target,target=/container/dir
  5. 捆绑挂载如果容器的那个目录非空，则内容被屏蔽，只显示宿主机的目录

Dockerfile命令

  1. ENTRYPOINT 和 CMD
    1. CMD ["command","param1","param2"]，如果没有ENTRYPOINT，就以CMD为默认命令
    2. CMD ["param1","param2"]，传递参数给ENTRYPOINT
    3. CMD command param1 param2，以shell的方式（/bin/sh -c）执行，ENTRYPOINT这样用也是一样
  2. COPY 和 ADD，ADD 多了一个功能，如果是src是url可以下载它
  3. WORKDIR 和 CD：CD 只对当前行命令有效，WORKDIR 则是整体改变目录
